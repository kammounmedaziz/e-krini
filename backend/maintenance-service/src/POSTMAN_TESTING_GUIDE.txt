3. POST /maintenance/add
Method: POST
URL: http://localhost:3007/maintenance/add
Description: Add new maintenance record with materials used. System auto-calculates coutMateriel from material prices and quantities, then applies 19.6% VAT.
Headers: 
  Content-Type: application/json
Body (raw JSON - Real Example from Database):
{
  "idVehicule": "69276ec4bd5a79fcd979a05e",
  "idMateriaux": [
    {
      "materiel": "6927718ad63e5dbddc493948",
      "quantiteUtilisee": 1
    },
    {
      "materiel": "692771cfd63e5dbddc49394c",
      "quantiteUtilisee": 15
    }
  ],
  "typeMaintenance": "Vidange",
  "dateMaintenance": "2024-11-20",
  "coutMainOeuvre": 50.00,
  "etat": "terminée",
  "description": "Vidange moteur + changement filtre à huile"
}

COST CALCULATION EXAMPLE:
- Material 1: Balais (BEG-001) 32€ × 1 = 32€
- Material 2: Batterie (BAT-60AH) 120€ × 15 = 1800€
- Subtotal Materials: 1832€
- Labor Cost (coutMainOeuvre): 50€
- Total HT: 1882€
- VAT (19.6%): 357.58€
- TOTAL TTC: 2239.58€

IMPORTANT: 
- idVehicule: Must be an existing vehicle ID (e.g., Peugeot 208 TN5678TU)
- idMateriaux: Array of materials used with their IDs and quantities
- System will populate material details and calculate costs automatically

REQUIRED FIELDS:
- idVehicule (string, valid MongoDB ObjectId of existing vehicle)
- typeMaintenance (string): Corrective, Révision, Réparation, Vidange, Contrôle technique
- dateMaintenance (date, ISO format YYYY-MM-DD, past date or today)
- coutMainOeuvre (number, 0-100000)

OPTIONAL FIELDS:
- etat (string): planifiée, en cours, terminée, annulée (default: planifiée)
- description (string, max 500 chars)
- idMateriaux (array of {materiel: ObjectId, quantiteUtilisee: number})

Response (Success):
  Status: 201 Created
  Body: Created maintenance record with populated vehicle and materials

TROUBLESHOOTING
================================================================================

Issue: "Cannot GET /maintenance/..."
Solution:
  - Verify server is running: Check terminal for startup messages
  - Verify URL is correct (check spelling and port 3007)
  - Verify the base path includes the module prefix (/maintenance, /materiel, or /vehicule)

Issue: "POST returns 400 Bad Request"
Solution:
  - Verify Content-Type header is set to "application/json"
  - Verify JSON body is valid (use Postman's JSON validator)
  - Verify all required fields are present in the request body
  - Check MongoDB is running and connected

Issue: "Connection refused"
Solution:
  - Verify server is running: node app.js in project directory
  - Verify port 3007 is not in use by another process
  - Run in PowerShell: netstat -ano | findstr :3007
  - Kill process if needed: taskkill /PID <PID> /F

Issue: "Database not connected"
Solution:
  - Verify MongoDB service is running
  - Check config/db.json for correct connection URL
  - Verify network connectivity to MongoDB

IMPORTANT NOTES
================================================================================

1. Server Status
   - Start server with: node app.js (from ProjectCrud directory)
   - Server runs on: http://localhost:3007
   - Confirm messages in terminal: Server startup confirmation and "Database connected"

2. Database
   - Uses MongoDB via Mongoose
   - Connection string from: config/db.json
   - Models located in: models/ directory

3. Route Prefixes
   - Maintenance routes: /maintenance
   - Material routes: /materiel
   - Vehicle routes: /vehicule
   - Full URLs: http://localhost:3007/<module>/...

4. Response Status Codes
   - 201 Created: Record successfully added (POST endpoints)
   - 200 OK: Data successfully retrieved or modified (GET, PUT, DELETE)
   - 400 Bad Request: Invalid input data
   - 404 Not Found: Resource not found
   - 500 Internal Server Error: Server error (DB connection issues)

5. Request Format
   - GET requests: Use URL parameters (/show, /showById/:id, etc.)
   - POST/PUT requests: Use JSON body with Content-Type: application/json header

6. Validation
   - Some POST/PUT endpoints require field validation
   - Check middleware in: middl/validateMaintenance.js
   - Ensure all required fields are provided in request body

7. Database Population & References (IMPORTANT)
   - Maintenance records POPULATE vehicle and material details
   - When creating maintenance, only send ObjectIds for idVehicule and idMateriaux
   - Response includes full nested objects with all details

8. Cost Calculation (Automatic)
   - coutMateriel = SUM(material.prixUnitaire × quantiteUtilisee)
   - coutTVA = (coutMainOeuvre + coutMateriel) × 19.6%
   - coutTotal = coutMainOeuvre + coutMateriel + coutTVA
